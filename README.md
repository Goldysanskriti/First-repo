Create a dynamic Plant Nursery Website suitable for a first-year (Semester 2) student.

Objective:
The website should act as a simple plant nursery platform where users can view, add, and explore plants available for sale, while also tracking plant details dynamically on the page.

Important Constraints:
- Use only HTML, CSS, and JavaScript
- Do NOT use backend, database, login/signup system, or external APIs
- Keep the design simple and beginner-friendly
- Use JavaScript arrays/objects for dynamic data
- Data will be temporary (lost on refresh)

Core Features:

1. Home Section:
- Display title "Green Nursery 🌿"
- Add a short description about buying plants and promoting greenery

2. Add Plant Form (Admin-like section):
Allow user to input plant details:
- Plant Name
- Plant Type (Indoor, Outdoor, Medicinal, Flowering, etc.)
- Price
- Available Quantity
- Location (Nursery Area)
- Status (Available / Sold Out)
- Optional: Image URL

3. Dynamic Plant Listing (Main Feature):
- Display all plants in a card layout (like shopping items)
- Each card should show:
  • Plant Name
  • Type
  • Price
  • Status
  • Small image (if provided)

4. View Details:
- Add a "View Details" button
- On click, show full plant details in a popup/modal or separate section:
  • Name
  • Type
  • Price
  • Quantity
  • Location
  • Status

5. Filter/Search Feature:
- Allow filtering by:
  • Plant Type
  • Availability (Available/Sold Out)
- Optional: simple search bar for plant name

6. Add to Cart (Simple Simulation):
- Add a button "Add to Cart"
- When clicked:
  • Show alert "Added to cart"
  • Reduce quantity (optional simple logic)

7. Delete Plant:
- Provide delete button for each plant (for management purpose)

8. Total Plants Counter:
- Show total number of plants available
- Optionally show total available vs sold out

9. UI/Design Requirements:
- Use green/nature theme 🌱
- Card-based layout (simple grid)
- Add hover effects for cards
- Keep layout clean and responsive (basic)

10. Code Requirements:
- Write everything in a single HTML file
- Use internal CSS and JavaScript
- Add comments explaining each section clearly
- Keep code simple and easy to understand for beginners

Output:
- Provide complete working code in one file
- Ensure it runs directly in browser without setup
